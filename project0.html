Problem

In cold-climate microgrids, heat is mission-critical. CHP units often run “thermal-first,” and electricity is a byproduct. Two issues often break day-ahead schedules:

  • Electric–thermal dependence is ignored: winter electric and thermal peaks occur together, so independent sampling underestimates joint stress.
  • Thermal slack is penalized asymmetrically: if only unmet heat is penalized, the optimizer can “run hot,” wasting fuel and increasing cycling.

What This Project Adds

Gaussian Copula Scenario Engine

Generates joint electric–thermal daily scenarios that preserve same-hour correlation and temporal persistence (AR(1)), producing realistic winter co-movement.

Symmetric Thermal Penalties

Uses two-sided slack variables to price both under-produced and over-produced heat, eliminating “run-hot” bias while keeping feasibility.

Case Study

Validation uses the Tok School microgrid (Alaska): biomass CHP (thermal-led), rooftop PV, lithium-ion BESS, and a grid interconnection with import price and zero export credit.

Methodology

1) Copula-Based Joint Scenarios

  • Use PIT to map electric/thermal loads to uniforms.
  • Map to latent Gaussian space; enforce same-hour E↔T correlation.
  • Apply AR(1) structure across hours to keep temporal realism.
  • Sample many daily trajectories and invert back to original marginals.

2) Thermal-Led MILP Dispatch

  • Electric balance from CHP + PV + BESS + grid import.
  • BESS SOC recursion + bounds + charge/discharge exclusivity.
  • Battery degradation cost via convex throughput proxy.
  • Grid import/export gated; export credit set to zero.
  • Robust objective: minimize worst-case (minimax) cost across scenarios.

3) Symmetric Thermal Penalties (No “Run-Hot” Bias)

Thermal adequacy uses two slacks: under-heat and over-heat. Both are priced, so the optimizer has no incentive to waste heat to avoid shortage penalties.
